# SPARK - 基于3R方法论的文本挖掘平台

## 项目概述

SPARK是一个基于**3R方法论**（Rewrite、Retrieve、Reformat）的智能文本挖掘平台，专门用于加速科学研究的文献综述流程。该平台专注于从学术论文中自动提取和处理固体聚合物电解质（SPEs）和电池材料相关的研究数据，能显著提升科研工作者的文献分析效率。

## 核心架构与技术栈

### 技术架构

- **编程语言**: Python 3.11
- **环境管理**: Conda环境 "ailab"
- **模型框架**: 基于Transformers的BERT模型
- **搜索引擎**: FAISS向量搜索
- **API集成**: DeepSeek-v3, OpenAI GPT系列模型
- **知识图谱**: NetworkX图形网络

## 核心模块详解

### 1. 主程序模块 (main.py:1-358)

**功能**: 平台的主要调度器，协调各个模块的工作流程

**核心特性**:
- 集成DeepSeek API进行化学知识提取
- 批量处理PDF文档（默认批次大小: 1，最大处理文件数: 150）
- 实现JSON格式数据的结构化提取与存储
- 支持错误处理和异常恢复机制

**工作流程**:
1. 从指定目录加载PDF文档
2. 批量处理文档并进行信息提取
3. 生成标准化JSON格式输出
4. 保存处理结果并更新记录

### 2. 化学助手模块 (modelchat.py:1-65)

**功能**: 专门用于化学领域知识提取的LLM助手

**技术特点**:
- 基于DeepSeek-v3模型的化学专业知识提取
- 内置重试机制（最多3次重试，延迟8秒）
- 针对化学领域的专用prompt优化
- 支持推理内容记录和分析

**使用方法**:
```python
assistant = ChatGPTChemicalAssistant(api_key, model)
response = assistant.generate_answer(prompt)
```

### 3. PDF检索模块 (pdfretrieve.py:1-50+)

**功能**: 实现PDF文档的智能检索和文本处理

**检索关键词体系**:
- **电解质系统**: Electrolyte, Solvent, Salt, Additive
- **电极材料**: Anode (石墨/锂金属阳极), Cathode (LFP, NCM系列)
- **电解质类型**: HCE, LHCE, DHCE, 氟化电解质, WSE
- **物化性质**: LUMO/HOMO能级、粘度、介电常数
- **性能指标**: 能量密度、库伦效率、循环寿命、倍率性能

### 4. 语义搜索模块 (semanticsearch.py)

**功能**: 基于BERT模型的语义相似度搜索

**技术实现**:
- 使用预训练BERT模型进行文本向量化
- FAISS索引构建和高效检索
- 支持大规模文档集合的快速语义搜索

### 5. 知识图谱构建模块

#### 图谱构建 (KG-construction.py:1-50+)

**功能**: 从提取数据构建化学知识图谱

**图谱结构**:
- **节点类型**: 聚合物、单体、盐类、引发剂
- **关系类型**: 组成关系、属性关系、性能关联
- **图结构**: 有向图（DiGraph）支持复杂关系表示

#### 图谱可视化 (KG-save_as_html.py)

**功能**: 将知识图谱导出为交互式HTML可视化

## 3R方法论实现

### Rewrite (查询重构)
- **功能**: 识别研究领域和关键术语
- **实现**: 针对化学领域的特定语言结构化prompt
- **特点**: 专注于电解质和电池材料领域术语

### Retrieve (文档检索)
- **功能**: PDF文本提取和预处理
- **实现**:
  - 使用PyMuPDF和pdfplumber进行PDF解析
  - 移除冗余内容（页眉、页脚）
  - 将文档组织成连贯的文本块
- **搜索能力**: 语义搜索和关键词检索相结合

### Reformat (数据重组)
- **功能**: 文本挖掘和信息结构化
- **实现**:
  - 生成标准化JSON格式数据
  - 构建知识图谱表示
  - 提供多种数据格式（JSON、CSV、HTML图谱）
  - 支持研究分析的标准格式

## 部署与使用

### 环境配置

```bash

conda env create -f environment.yml
conda activate ailab

# 安装依赖
pip install -r requirements.txt
```

1. **API配置**: 在main.py中配置DeepSeek API密钥
2. **路径设置**: 设置PDF文档目录路径
3. **参数调整**: 根据需要调整批处理大小和文件限制

### 执行流程

```bash
python main.py
```

**处理步骤**:
1. 扫描指定目录下的PDF文件
2. 批量处理文档并提取文本
3. 使用LLM进行信息提取和结构化
4. 生成JSON格式的结构化数据
5. 更新处理记录和错误日志
